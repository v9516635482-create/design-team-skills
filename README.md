# Design Team Skills

Плагин для Claude Code с четырьмя агентами под дизайн-проекты в Figma.

## Что внутри

| Агент | Зачем | Когда звать |
|---|---|---|
| **design-advisor** | UX-советник и информационный архитектор: проектирует раскладку экрана, решает, где разместить элементы и какой контрол выбрать — по UX-правилам (Фиттс, Хик, Якоб, гештальт) | «куда поставить X», «спроектируй структуру экрана», «наведи порядок в раскладке» |
| **design-qa** | Тестировщик макетов: доступность (контраст, alt, размеры зон), адаптивность, юзабилити | «проверь макет», «нормально ли на мобиле», «найди проблемы» |
| **design-reviewer** | Критик дизайна: иерархия, типографика, цвет, композиция, впечатление | «отревьюй дизайн», «как сделать солиднее», «раскритикуй» |
| **design-researcher** | Ресёрчер: референсы, тренды, конкуренты, UX-паттерны | «найди примеры», «что в тренде», «как делают другие» |

## Общая база — свод UX-правил

Все четыре агента перед работой читают общий свод UX-правил (`rules/ux-rules.md`): законы Фиттса, Хика, Якоба, гештальт-принципы, паттерны размещения и выбора контролов, анти-паттерны. На него они и опираются — каждый вывод (советник, QA, ревьюер, ресёрчер) привязан к конкретному правилу, а не к вкусовщине. Это общий фундамент команды, а не персональная база одного агента.

## Требования

- **Claude Code** с поддержкой плагинов.
- Для `design-qa` и `design-reviewer` нужна подключённая **Figma** (MCP) — они читают скриншоты и контекст макета. Без Figma эти агенты работают только со свёрстанными файлами (HTML/CSS).
- `design-researcher` использует веб-поиск и Figma (Figma — опционально).
- `design-advisor` работает с макетами Figma, свёрстанными страницами или задачей «с нуля».

## Установка

```
/plugin marketplace add ux-vika/design-team-skills
/plugin install design-team-skills@design-team-marketplace
```

После установки перезапусти Claude Code. Агенты вызываются автоматически по смыслу задачи
или явно — по имени.
